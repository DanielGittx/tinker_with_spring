NB:- Roles should start with "ROLES_XXXXXX" IN Spring security and expressed on antMatchers as "XXXXXX"
E.g 
     -  In database :- ROLE_MODERATOR  
     -  Expressed as:- .antMatchers("/fetch/**").hasRole("MODERATOR")  in WebSecurityConfig implementation
